    2.3Static Semantics: ContainsDuplicateLabels

    With parameter labelSet.

    TryStatement:try(Expression)Block TryStatement:try(varVariableDeclarationList)Block TryStatement:try(LetOrConstBindingList)Block
    1. Return ContainsDuplicateLabels of Block with argument labelSet.
    TryStatement:tryBlockCatch TryStatement:try(Expression)BlockCatch TryStatement:try(varVariableDeclarationList)BlockCatch TryStatement:try(LetOrConstBindingList)BlockCatch
    1. Let hasDuplicates be ContainsDuplicateLabels of Block with argument labelSet.
    2. If hasDuplicates is true, return true.
    3. Return ContainsDuplicateLabels of Catch with argument labelSet.
    TryStatement:tryBlockFinally TryStatement:try(Expression)BlockFinally TryStatement:try(varVariableDeclarationList)BlockFinally TryStatement:try(LetOrConstBindingList)BlockFinally
    1. Let hasDuplicates be ContainsDuplicateLabels of Block with argument labelSet.
    2. If hasDuplicates is true, return true.
    3. Return ContainsDuplicateLabels of Finally with argument labelSet.
    TryStatement:tryBlockCatchFinally TryStatement:try(Expression)BlockCatchFinally TryStatement:try(varVariableDeclarationList)BlockCatchFinally TryStatement:try(LetOrConstBindingList)BlockCatchFinally
    1. Let hasDuplicates be ContainsDuplicateLabels of Block with argument labelSet.

    2.9Runtime Semantics: TryWithResourcesInstantiation (boundName, declarationKind, tryEnv)

    The abstract operation TryWithResourcesInstantiation with arguments boundName, declarationKind and tryEnv performs the following steps:

    1. Assert: tryEnv is a Lexical Environment.
    2. Assert: Type(boundName) is String or Undefined.
    3. Assert: declarationKind is "none", "var", "let" or "const".
    4. If declarationKind is "const" or "let", then
      1. Assert: Type(boundName) is String.
      2. Let envRec be tryEnv's EnvironmentRecord.
      3. Assert: envRec is a declarative EnvironmentRecord.
      4. If declarationKind is "const", then
        1. Perfom ! envRec.CreateImmutableBinding(boundName, true).
      5. Else,
        1. Perfom ! envRec.CreateMutableBinding(boundName, false).

    2.10Runtime Semantics: TryResourceEvaluation (resource, boundName, declarationKind)

    The abstract operation TryResourceEvaluation with arguments resource, boundName and declarationKind performs the following steps:

    1. Assert: declarationKind is "none", "var", "let", or "const".
    2. If declarationKind is "none", then
      1. Assert: resource is an Expression.
      2. Assert: boundName is undefined.
      3. Let exprRef be the result of evaluating resource.
      4. Return ? GetValue(exprRef).
    3. Else,
      1. Assert: resource is a VariableDeclaration or a LexicalBinding.
      2. Assert: Type(boundName) is String.
      3. Let decl be the result of evaluating resource.
      4. ReturnIfAbrupt(decl).
      5. Return ? ResolveBinding(boundName).

    2.11Runtime Semantics: TryWithResourcesEvaluation (resources, boundNames, declarationKind, depth)

    With arguments resources, boundNames, declarationKind and depth.

    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(varVariableDeclarationList)BlockCatchoptFinallyopt TryStatement:try(LetOrConstBindingList)BlockCatchoptFinallyopt
    1. Assert: resources is a non-empty List.
    2. Assert: boundNames is a List.
    3. Assert: declarationKind is "none", "var", "let", or "const".
    4. Let maxDepth be the number of elements in resources.
    5. Assert: 0 ≤ depth < maxDepth.
    6. Let resource be the element at index depth of resources.
    7. If declarationKind is "none", then
      1. Let boundName be undefined.
    8. Else,
      1. Assert: The number of elements in boundNames is equal to maxDepth.
      2. Let boundName be the element at index depth of boundNames.
    9. Let oldEnv be the running execution context's LexicalEnvironment.
    10. Let tryEnv be NewDeclarativeEnvironment(oldEnv).
    11. Perform TryWithResourcesInstantiation(boundName, declarationKind, tryEnv).
    12. Set the running execution context's LexicalEnvironment to tryEnv.
    13. Let exprCompletion be TryResourceEvaluation(resource, boundName).
    14. If exprCompletion.[[Type]] is not normal, then
      1. Set the running execution context's LexicalEnvironment to oldEnv.
      2. Return exprCompletion.
    15. If depth < maxDepth - 1, then
      1. Let result be TryWithResourcesEvaluation(resources, boundNames, declarationKind, depth + 1).
    16. Else,
      1. Let result be the result of evaulating Block.
    17. Let expr be exprCompletion.[[Value]].
    18. If expr is null or undefined, then
      1. Let cleanupResult be NormalCompletion(empty).
    19. Else,
      1. Let cleanupResult be Invoke(expr, @@dispose).
    20. Set the running execution context's LexicalEnvironment to oldEnv.
    21. If cleanupResult.[[Type]] is throw, then
      1. If result.[[Type]] is throw, then
        1. Let errors be CreateArrayFromList(« result.[[Value]], cleanupResult.[[Value]] »).
        2. Throw a new AggregateError exception the errors in errors.
      2. Else,
        1. Return cleanupResult.
    22. Else,
      1. Assert: cleanupResult.[[Type]] is normal.
    23. Return result.

    2.12Runtime Semantics: Evaluation

    TryStatement:try(Expression)Block
    1. Let resources be « Expression ».
    2. Return ? TryWithResourcesEvaluation(resources, « », "none", 0).
    TryStatement:try(Expression)BlockCatch
    1. Let resources be « Expression ».
    2. Let B be TryWithResourcesEvaluation(resources, « », "none", 0).
    3. If B.[[Type]] is throw, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    4. Else, let C be B.
    5. Return Completion(UpdateEmpty(C, undefined)).
    TryStatement:try(Expression)BlockFinally
    1. Let resources be « Expression ».
    2. Let B be TryWithResourcesEvaluation(resources, « », "none", 0).
    3. Let F be the result of evaluating Finally.
    4. If F.[[Type]] is normal, set F to B.
    5. Return Completion(UpdateEmpty(F, undefined)).
    TryStatement:try(Expression)BlockCatchFinally
    1. Let resources be « Expression ».
    2. Let B be TryWithResourcesEvaluation(resources, « », "none", 0).
    3. If B.[[Type]] is throw, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    4. Else, let C be B.
    5. Let F be the result of evaluating Finally.
    6. If F.[[Type]] is normal, set F to C.
    7. Return Completion(UpdateEmpty(F, undefined)).

    3. Let B be TryWithResourcesEvaluation(resources, boundName, "var", 0).

    5. Return ? TryWithResourcesEvaluation(resources, boundName, declarationKind, 0).
    TryStatement:try(LetOrConstBindingList)BlockCatch
    1. Let resources be TryResources of BindingList.
    2. Let boundNames be the BoundNames of BindingList.
    3. If IsConstantDeclaration of LetOrConst is true, let declarationKind be "const".
    4. Else, let declarationKind be "let".
    5. Let B be TryWithResourcesEvaluation(resources, boundName, declarationKind, 0).
    6. If B.[[Type]] is throw, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    7. Else, let C be B.
    8. Return Completion(UpdateEmpty(C, undefined)).
    TryStatement:try(LetOrConstBindingList)BlockFinally
    1. Let resources be TryResources of BindingList.
    2. Let boundNames be the BoundNames of BindingList.
    3. If IsConstantDeclaration of LetOrConst is true, let declarationKind be "const".
    4. Else, let declarationKind be "let".
    5. Let B be TryWithResourcesEvaluation(resources, boundName, declarationKind, 0).
