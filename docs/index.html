    2.1The try Statement

    Syntax

    TryWithResourcesDeclaration[Yield, Await]:constBindingList[+In, ?Yield, ?Await] TryStatement[Yield, Await, Return]:try([lookahead ∉ { let [ }]Expression[+In, ?Yield, ?Await])Block[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]optFinally[?Yield, ?Await, ?Return]opt try(TryWithResourcesDeclaration[?Yield, ?Await])Block[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]optFinally[?Yield, ?Await, ?Return]opt tryBlock[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return] tryBlock[?Yield, ?Await, ?Return]Finally[?Yield, ?Await, ?Return] tryBlock[?Yield, ?Await, ?Return]Catch[?Yield, ?Await, ?Return]Finally[?Yield, ?Await, ?Return]

    2.2Static Semantics: Early Errors

    TryWithResourcesDeclaration:constBindingList
    • It is a Syntax Error if any element of the BoundNames of BindingList contains "let".
    • It is a Syntax Error if any element of the BoundNames of BindingList contains any duplicate entries.
    • It is a Syntax Error if any element of the BoundNames of BindingList also occurs in the VarDeclaredNames of Block.

    2.3Static Semantics: ContainsDuplicateLabels

    With parameter labelSet.

    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(TryWithResourcesDeclaration)BlockCatchoptFinallyopt
    1. Let hasDuplicates be ContainsDuplicateLabels of Block with argument labelSet.
    2. If hasDuplicates is true, return true.
    3. If Catch is present, then
      1. Set hasDuplicates to ContainsDuplicateLabels of Catch with argument labelSet.
      2. If hasDuplicates is true, return true.
    4. If Finally is present, then
      1. Set hasDuplicates to ContainsDuplicateLabels of Finally with argument labelSet.
      2. If hasDuplicates is true, return true.
    5. Return false.

    2.4Static Semantics: ContainsUndefinedBreakTarget

    With parameter labelSet.

    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(TryWithResourcesDeclaration)BlockCatchoptFinallyopt
    1. Let hasUndefinedTarget be ContainsUndefinedBreakTarget of Block with argument labelSet.
    2. If hasUndefinedTarget is true, return true.
    3. If Catch is present, then
      1. Set hasUndefinedTarget to ContainsUndefinedBreakTarget of Catch with argument labelSet.
      2. If hasUndefinedTarget is true, return true.
    4. If Finally is present, then
      1. Set hasUndefinedTarget to ContainsUndefinedBreakTarget of Finally with argument labelSet.
      2. If hasUndefinedTarget is true, return true.
    5. Return false.

    2.5Static Semantics: ContainsUndefinedContinueTarget

    With parameters iterationSet and labelSet.

    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(TryWithResourcesDeclaration)BlockCatchoptFinallyopt
    1. Let hasUndefinedTarget be ContainsUndefinedContinueTarget of Block with argument labelSet.
    2. If hasUndefinedTarget is true, return true.
    3. If Catch is present, then
      1. Set hasUndefinedTarget to ContainsUndefinedContinueTarget of Catch with argument labelSet.

    4. If Finally is present, then
      1. Set hasUndefinedTarget to ContainsUndefinedContinueTarget of Finally with argument labelSet.
      2. If hasUndefinedTarget is true, return true.
    5. Return false.

    2.6Static Semantics: BoundNames

    TryWithResourcesDeclaration:constBindingList
    1. Return the BoundNames of BindingList.

    2.7Static Semantics: IsConstantDeclaration

    TryWithResourcesDeclaration:constBindingList
    1. Return true.

    2.8Static Semantics: VarDeclaredNames

    TryStatement:try(Expression)BlockCatchoptFinallyopt TryStatement:try(TryWithResourcesDeclaration)BlockCatchoptFinallyopt
    1. Let names be VarDeclaredNames of Block.
    2. If Catch is present, then
      1. Append to names the elements of the VarDeclaredNames of Catch.
    3. If Finally is present, then
      1. Append to names the elements of the VarDeclaredNames of Finally.
    4. Return names.

    2.9Static Semantics: VarScopedDeclarations

    2. If Catch is present, then
      1. Append to declarations the elements of the VarScopedDeclarations of Catch.
    3. If Finally is present, then
      1. Append to declarations the elements of the VarScopedDeclarations of Finally.
    4. Return declarations.

    2.10Static Semantics: TryResources

    TryWithResourcesDeclaration:constBindingList
    1. Return TryResources of BindingList.
    BindingList:BindingList,LexicalBinding
    1. Let resources be TryResources of BindingList.
    2. Append to resources the elements of the TryResources of LexicalBinding.
    3. Return resources.
    BindingList:LexicalBinding
    1. Return « LexicalBinding ».

    2.11Runtime Semantics: TryResourceEvaluation

    LexicalBinding:BindingIdentifierInitializer
    1. Let bindingId be StringValue of BindingIdentifier.
    2. Let lhs be ResolveBinding(bindingId).
    3. If IsAnonymousFunctionDefinition(Initializer) is true, then
      1. Let value be the result of performing NamedEvaluation for Initializer with argument bindingId.
    4. Else,
      1. Let rhs be the result of evaluating Initializer.
      2. Let value be ? GetValue(rhs).
    5. Perform ! InitializeReferencedBinding(lhs, value).
    6. Return « value »
    LexicalBinding:BindingPatternInitializer
    1. Let rhs be the result of evaluating Initializer.
    2. Let value be ? GetValue(rhs).
    3. Let env be the running execution context's LexicalEnvironment.
    4. Perform BindingInitialization for BindingPattern using value and env as the arguments.
    5. Return « value »

    2.12Runtime Semantics: Dispose ( obj )

    The abstract operation Dispose with argument obj performs the following steps:

    1. If obj is null or undefined, return undefined.
    2. If ! GetFunctionKind() is "async" or "async generator", then
      1. Let asyncMethod be ? GetMethod(obj, @@asyncDispose).
      2. If asyncMethod is not undefined, then
        1. Let asyncResult be ? Call(asyncMethod, obj).
        2. Return ? Await(asyncResult).
    3. Let syncMethod be ? GetMethod(obj, @@dispose).
    4. Return ? Call(syncMethod, obj).

    2.13Runtime Semantics: TryWithResourcesInstantiation (boundNames, tryEnv)

    The abstract operation TryWithResourcesInstantiation with arguments boundNames and tryEnv performs the following steps:

    1. Assert: boundNames is List of String values.
    2. Assert: tryEnv is a Lexical Environment.
    3. Let envRec be tryEnv's EnvironmentRecord.
    4. Assert: envRec is a declarative EnvironmentRecord.
    5. For each boundNames of boundNames, do
      1. Perfom ! envRec.CreateImmutableBinding(boundName, true).

    2.14Runtime Semantics: TryWithResourcesEvaluation (resources, Block)

    The abstract operation TryWithResourcesEvaluation with arguments resources and Block performs the following steps:

    1. Assert: resources is a non-empty List.
    2. Assert: Each element of resources is either an Expression or LexicalBinding.
    3. Assert: Block is a Block.
    4. Let resourcesLength be the number of elements in resources.
    5. Let resourceStack be a new empty List.
    6. Let lastCompletion be NormalCompletion(empty).
    7. Let n be 0.
    8. Repeat, while lastCompletion.[[Type]] is normal and n < resourcesLength
      1. Let resource be the nth element of resources.
      2. If resource is an Expression, then
        1. Let lastCompletion be the result of evaluating resource.
      3. Else,
        1. Let lastCompletion be the result of performing TryResourceEvaluation for resource.
      4. If lastCompletion.[[Type]] is normal, then
        1. Add lastCompletion.[[Value]] as the last element of resourceStack.
        2. Set n to n + 1.
    9. If lastCompletion.[[Type]] is normal, then
      1. Let lastCompletion be the result of evaluating Block.
    10. Let errors be a new empty List.
    11. If lastCompletion.[[Type]] is throw, then
      1. Add lastCompletion.[[Value]] as the last element of errors.
    12. For each resource of resourceStack, in reverse list order, do
      1. Set disposeCompletion to Dispose(resource).
      2. Assert: disposeCompletion.[[Type]] is either throw or normal.
      3. If disposeCompletion.[[Type]] is throw, then
        1. Add disposeCompletion.[[Value]] as the last element of errors.
    13. Let errorsLength be the number of elements in errors.
    14. If errorsLength = 0, return lastCompletion.
    15. If errorsLength = 1, then
      1. Let error be the first element of errors.
      2. return ThrowCompletion(error).
    16. Let errorsArray be CreateArrayFromList(errors).
    17. Throw a new AggregateError exception with the errors in errorsArray.

    2.15Runtime Semantics: Evaluation

    TryStatement:try(Expression)BlockCatchoptFinallyopt
    1. Let resources be « Expression ».
    2. Let B be TryWithResourcesEvaluation(resources, Block).
    3. If B.[[Type]] is throw and Catch is present, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    4. Else, let C be B.
    5. If Finally is present, let F be the result of evaluating Finally.
    6. Else, let F be C.
    7. If F.[[Type]] is normal, set F to C.
    8. Return Completion(UpdateEmpty(F, undefined)).
    TryStatement:try(TryWithResourcesDeclaration)BlockCatchoptFinallyopt
    1. Let resources be the TryResources of TryWithResourcesDeclaration.
    2. Let boundNames be the BoundNames of TryWithResourcesDeclaration.
    3. Let oldEnv be the running execution context's LexicalEnvironment.
    4. Let tryEnv be ! NewDeclarativeEnvironment(oldEnv).
    5. Perform ! TryWithResourcesInstantiation(boundNames, tryEnv).
    6. Set the running execution context's LexicalEnvironment to tryEnv.
    7. Let B be TryWithResourcesEvaluation(resources, Block).
    8. Set the running execution context's LexicalEnvironment to oldEnv.
    9. If B.[[Type]] is throw and Catch is present, let C be CatchClauseEvaluation of Catch with argument B.[[Value]].
    10. Else, let C be B.
    11. If Finally is present, let F be the result of evaluating Finally.
    12. Else, let F be C.
    13. If F.[[Type]] is normal, set F to C.
    14. Return Completion(UpdateEmpty(F, undefined)).

    3Fundamental Objects

    3.1Error Objects

    3.1.1AggregateError Objects

    An AggregateError Object is an extended NativeError object that encapsulates multiple errors.

    Note
    NOTE: This is based on the same design for AggregateError in the Promise.any proposal.

    3.1.1.1The AggregateError Constructor

    The AggregateError constructor:

    • is the intrinsic object %AggregateError%
    • is the initial value of the AggregateError property of the global object.
    • creates and initializes a new AggregateError object when called as a function rather than as a constructor. Thus the function call AggregateError(...) is equivalent to the object creation expression new AggregateError(...) with the same arguments.
    • is designed to be subclassable. It may be used as the value of an extends clause of a class definition. Subclass constructors that intend to inherit the specified AggregateError behaviour must include a super call to the AggregateError constructor to create and initialize subclass instances with an [[ErrorData]] internal slot.

    3.1.1.1.1AggregateError ( errors, message )

    When the AggregateError function is called with arguments errors and message, the following steps are taken:

    1. If NewTarget is undefined, let newTarget be the active function object; else let newTarget be NewTarget.
    2. Let O be ? OrdinaryCreateFromConstructor(newTarget, "%AggregateErrorPrototype%", « [[ErrorData]] »).
    3. Let errorsIterator be ? GetMethod(errors, @@iterator).
    4. Let errorsList be ? IterableToList(errors, errorsIterator).
    5. Let errorsArray be ! CreateArrayFromList(errorsList).
    6. Perform ! CreateDataProperty(O, "errors", errorsArray).
    7. If message is not undefined, then
      1. Let msg be ? ToString(message).
      2. Let msgDesc be the PropertyDescriptor { [[Value]]: msg, [[Writable]]: true, [[Enumerable]]: false, [[Configurable]]: true }.
      3. Perform ! DefinePropertyOrThrow(O, "message", msgDesc).
    8. Return O.

    3.1.1.2Properties of the AggregateError Constructor

    The AggregateError constructor:

    • has a [[Prototype]] internal slot whose value is the intrinsic object %Error%.
    • has the following properties:

    3.1.1.2.1AggregateError.prototype

    The intial value of AggregateError.prototype is the intrinsic object %AggregateErrorPrototype%.

    This property has the attributes { [[Writable]]: false, [[Enumerable]]: false, [[Configurable]]: false }.

    3.1.1.3Properties of the AggregateError Prototype Object

    The AggregateError prototype object:

    • is an ordinary object.
    • is not an Error instance and does not have an [[ErrorData]] internal slot.
    • has a [[Prototype]] internal slot whose value is the intrinsic object %ErrorPrototype%.

    3.1.1.3.1AggregateError.prototype.constructor

    The initial value of AggregateError.prototype.constructor is the intrinsic %AggregateError% object.

    3.1.1.3.2AggregateError.prototype.message

    The initial value of AggregateError.prototype.message is the empty String.

    3.1.1.3.3AggregateError.prototype.name

    The initial value of AggregateError.prototype.name is "AggregateError".

    3.1.1.3.4AggregateError.prototype.errors

    The initial value of AggregateError.prototype.errors is undefined.

    3.1.1.4Properties of AggregateError Instances

    AggregateError instances are ordinary objects that inherit properties from the AggregateError prototype object and have an [[ErrorData]] internal slot whose value is undefined. The only specified uses of [[ErrorData]] is to identify Error, AggregateError, and NativeError instances as Error objects within Object.prototype.toString.
