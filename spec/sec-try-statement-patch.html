Static Semantics: Early Errors

TryUsingDeclaration : `const` BindingList
  • It is a Syntax Error if any element of the BoundNames of |BindingList| contains `"let"`.
  • It is a Syntax Error if any element of the BoundNames of |BindingList| contains any duplicate entries.
  • It is a Syntax Error if any element of the BoundNames of |BindingList| also occurs in the VarDeclaredNames of |Block|.

Runtime Semantics: TryUsingBodyEvaluation (_Head_, _boundNames_, _Block_, _hint_)

The abstract operation TryUsingBodyEvaluation with arguments _Head_, _boundNames_, _Block_, and _hint_ performs the following steps:

1. Assert: _Head_ is either an |Expression| or |TryUsingDeclaration|. 1. Assert: _boundNames_ is a List of String values. 1. Assert: _Block_ is a |Block|. 1. Assert: _hint_ is either ~sync~ or ~async~. 1. Let _oldEnv_ be the running execution context's LexicalEnvironment. 1. Let _tryEnv_ be ! NewTryUsingEnvironment(_oldEnv_, _hint_). 1. Let _envRec_ be _tryEnv_'s EnvironmentRecord. 1. For each _boundNames_ of _boundNames_, do 1. Perform ! _envRec_.CreateImmutableBinding(_boundName_, *true*). 1. Let _completion_ be the result of evaluating _Head_. 1. If _completion_.[[Type]] is ~normal~, then 1. If _Head_ is an |Expression|, then 1. Set _completion_ to _envRec_.AddDisposableResource(_completion_.[[Value]]). 1. Else, _Head_ is a |TryUsingDeclaration| parse node and disposable resources were added when bindings were initialized. 1. If _completion_.[[Type]] is ~normal~, then 1. Let _completion_ be the result of evaluating _Block_. 1. Let _errors_ be a new empty List. 1. If _completion_.[[Type]] is ~throw~, then 1. Append _completion_.[[Value]] to _errors_. 1. For each _resource_ of _envRec_.[[ResourceStack]], in reverse list order, do 1. Let _result_ be Dispose(_resource_.[[ResourceValue]], _hint_, _resource_.[[DisposeMethod]]). 1. If _result_.[[Type]] is ~throw~, then 1. Append _result_.[[Value]] to _errors_. 1. Set the running execution context's LexicalEnvironment to _oldEnv_. 1. Let _errorsLength_ be the number of elements in _errors_. 1. If _errorsLength_ > 1, then 1. Let _error_ be a newly created `AggregateError` object. 1. Perform ! DefinePropertyOrThrow(_error_, *"errors"*, PropertyDescriptor { [[Configurable]]: *true*, [[Enumerable]]: *false*, [[Writable]]: *true*, [[Value]]: ! CreateArrayFromList(_errors_) }). 1. Return ThrowCompletion(_error_). 1. If _errorsLength_ = 1, then 1. Let _error_ be the first element of _errors_. 1. Return ThrowCompletion(_error_). 1. Return _completion_.

Runtime Semantics: CatchFinallyEvaluation (_tryCompletion_, _Catch_, _Finally_)

The abstract operation CatchFinallyEvaluation with arguments _tryCompletion_, _Catch_, and _Finally_ performs the following steps:

1. Assert: _tryCompletion_ is a Completion Record. 1. Assert: _Catch_ is either a |Catch| or is *undefined*. 1. Assert: _Finally_ is either a |Finally| or is *undefined*. 1. If _tryCompletion_.[[Type]] is ~throw~ and _Catch_ is not *undefined*, let _C_ be CatchClauseEvaluation of _Catch_ with argument _tryCompletion_.[[Value]]. 1. Else, let _C_ be _tryCompletion_. 1. If _Finally_ is not *undefined*, let _F_ be the result of evaluating _Finally_. 1. Else, let _F_ be _C_. 1. If _F_.[[Type]] is ~normal~, set _F_ to _C_. 1. Return Completion(UpdateEmpty(_F_, *undefined*)).
